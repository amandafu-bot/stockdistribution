# PRD - Stock Distribution Tool
# 📄 電商庫存分配小工具 PRD（前端 Demo 版本）

## 1️⃣ 專案目標（Goal）

建立一個 **簡易的庫存分配工具（有介面）**，讓使用者在每月初：

1. 上傳 Excel（商品 + 當月可分配總庫存）
2. 設定每個 SKU 的分配規則
3. 系統自動計算：
    - 蝦皮
    - Momo
    - 官網
        
        各平台應配置庫存
        
4. 匯出結果 Excel

👉 本版本重點為：

- UI 操作流程
- 規則設定
- 計算結果顯示
    
    （不需串接平台 API）
    

---

## 2️⃣ 使用者情境（User Scenario）

### 👤 使用者

電商店主（不熟程式）

### 📅 每月流程

1. 匯出 ERP 或庫存 Excel
2. 上傳到工具
3. 調整本月策略（例：衝流量 / 穩定販售）
4. 下載結果並手動上架各平台

---

## 3️⃣ 系統核心概念

### SKU 為最小單位

每個 SKU：

- 必須選擇：
    - ✅ 一個主條件
    - ✅ 多個次條件（可選）

# 6️⃣ 主條件（Primary Condition）

## 🔷 Story 1｜穩定販售型（總比例 = 100%）

### 📘 說明

依固定比例分配庫存。

### 📊 UI 設計

當使用者選擇此主條件：

顯示比例設定：

| 平台 | 比例 |
| --- | --- |
| 蝦皮 | 50% |
| Momo | 30% |
| 官網 | 20% |

👉 必須加總 = 100%

### 📐 計算邏輯

```
平台庫存 = 總庫存 × 比例
```

---

## 🔷 Story 2｜衝流量型（總比例 > 100%）

### 📘 說明

用於衝高曝光與排名，允許超賣。

### 📊 UI 設計

比例可 > 100%

例如：

| 平台 | 比例 |
| --- | --- |
| 蝦皮 | 80% |
| Momo | 50% |
| 官網 | 30% |
| （總 = 160%） |  |

### 📐 計算邏輯

```
平台庫存 = 總庫存 × 比例
```

👉 顯示警示：

⚠️「總比例大於 100%，可能導致超賣」

---

# 7️⃣ 次條件（Secondary Condition）

可複選。

---

## 🔶 Story 3｜預扣（通路、商品型態）

### 📘 說明

先保留部分庫存。

### 📊 UI 設計

使用者可以新增多筆：

例：

| 類型 | 平台 | 數量 |
| --- | --- | --- |
| 直播 | 蝦皮 | 50 |
| 批發 | 官網 | 100 |

### 📐 計算順序

1. 先扣除預扣
2. 剩餘庫存再做主條件分配

```
可分配庫存 = 總庫存 – 預扣總量
```

---

## 🔶 Story 4｜人工介入（例外處理）

### 📘 說明

手動覆蓋某平台庫存。

### 📊 UI 設計

例：

| 平台 | 強制庫存 |
| --- | --- |
| Momo | 200 |

### 📐 計算邏輯

最後覆蓋：

```
若有人工介入 → 覆蓋該平台數值
```

---
